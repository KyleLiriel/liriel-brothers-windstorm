# Project Rules

- Keep the game prototype small, readable, and easy to iterate.
- Use Phaser 3, TypeScript, and Vite for browser gameplay.
- Use placeholder graphics made with Phaser shapes until real art is requested.
- Keep gameplay rules clear and avoid hiding puzzle logic inside rendering code.
- Do not add external assets, paid libraries, or network services without explicit approval.
- Keep controls documented in `README.md` whenever they change.
- Run `npm run build` before considering a change complete.
